# Консольная мини-игра-симулятор «Смулятор Перекупа»

---

## Введение

В процессе разработки проекта было принято осознанное решение отказаться от классической игровой модели с уроном, боями и ликвидацией противников.

Вместо этого игра смещена в сторону:

- дружелюбного игрового опыта,
- социально-экономической симуляции,
- моделирования решений и их последствий.

Такой подход позволил сделать процесс:

- менее конфликтным,
- более стратегическим,
- приближённым к реальным жизненным ситуациям.

Из-за смены тематики возникли небольшие отклонения от исходного ТЗ, которые постарался компенсировать повышенной сложностью реализации:

- введены вероятностные события,
- реализованы параллельные действия соперника,
- разработана система рисков и исходов,
- увеличена глубина логики игрового процесса.

Проект представляет собой экономический и поведенческий симулятор,
сохраняющий трудоемкость разработки и сложность механик.

---

# 1. Лор и концепция игры

Игрок выступает в роли автомеханика-перекупа энтузиаста, работающего на вторичном рынке автомобилей.

Вместо боев и конфликтов — здесь:

- переговоры и торговля,
- оценка рисков,
- восстановление машин,
- долгосрочные инвестиционные проекты.

Каждый ход — это стратегическое решение, принимаемое в условиях ограниченного бюджета и случайных событий.

---

## Ветка 1 — Переговоры с перекупом

Игрок выбирает стиль поведения:

- жёсткий торг по цене  
- быстрый выкуп  
- психологическое давление  
- уход и возврат позже  

Соперник может быть:

- спокойным продавцом  
- хитрым перекупом  
- агрессивным переговорщиком  

Результат рассчитывается числовой моделью
с учётом взаимодействия двух стратегий.

---

## Ветка 2 — Перепродажа автомобилей

Ветка моделирует рыночные сделки с автомобилями.

Игрок:

- покупает авто,
- оценивает риски и перспективу,
- принимает решение — рискнуть или отказаться.

Реализованы:

- удачные и неудачные покупки,
- скрытые риски,
- редкие удачные ситуации.

---

## Ветка 3 — Инвестиционный портфель проектов

Самая сложная и глубокая ветка.

Игрок управляет проектами:

- быстрые доработки,
- восстановительные ремонты,
- долгие крупные сборки / свапы.

Каждый проект имеет:

- длительность (много ходов),
- заморозку бюджета,
- вероятность событий (ускорение / задержка),
- потенциальную прибыль или убыток.

Параллельно работает соперник-инвестор, 
ведущий собственный набор проектов.

Игроку необходимо:

- распределять бюджет,
- выбирать время выхода,
- управлять риском.

---

# 2. Система достижений (артефактов)

В игре реализована персональная система достижений:

| Артефакт | Условие |
|---------|--------|
| Первая кровь | первая успешная сделка |
| Настоящий перекуп | 10 завершённых сделок |
| Жирный куш | прибыль > 100000 ₽ |
| Терпеливый механик | успешный долгий проект |
| На грани | досрочная продажа с риском |
| Космическая удача | редкое положительное событие |

Особенности реализации:

- достижения **не влияют на баланс игры**
- выполняют роль прогресс-индикаторов
- сохраняются **для каждого пользователя отдельно**
- активируются при входе в аккаунт

# 3. Инструкция по использованию

---

## Запуск игры

Требуется Python **3.10+**

```bash
python main.py
```

---

## Работа с пользователями

Поддерживается:

- регистрация аккаунта
- вход в профиль 

Сохраняется:
- прогресс игрока
- персональные достижения

---
## Геймплейный цикл

1. Выбор сюжетной ветки
2. Выполнение игровых действий 
3. Получение результата сделки 
4. Изменение бюджета 
5. Проверка условий победы / поражения 
6. Возможная выдача достижений

**Игра завершается при:**

- достижении финансовой цели, 
- банкротстве, 
- принудительном выходе игрока.

# 4. Техническое описание проекта
Структура проекта
```
project/
 ├─ main.py                — главный сценарий
 ├─ auth.py                — система логинов и сессий
 ├─ player.py              — модель игрока и соперника
 ├─ save_system.py         — загрузка и сохранение прогресса
 ├─ artifacts.py           — описание артефактов
 ├─ artifacts_hooks.py     — логика выдачи достижений
 ├─ artifact_storage.py    — файловое хранилище артефактов
 ├─ branch1_basic.py       — ветка переговоров
 ├─ branch2_market.py      — ветка перепродажи
 ├─ branch3_portfolio.py   — ветка инвестиционных проектов
 └─ storage/               — пользовательские данные
```

---
## Архитектурные решения

* модульная структура 
* каждая ветка — отдельный игровой модуль 
* соперник реализован как независимый объект 
* система артефактов декомпозирована на:
  - описание достижений, 
  - триггеры-события, 
  - файловое хранилище. 
* прогресс не зависит от ветки и сохраняется глобально 
* достижения привязаны к имени пользователя 
* предусмотрена возможность расширения проекта

---
## Технические особенности реализации

* параллельная симуляция игрока и соперника 
* управление состоянием портфеля проектов 
* вероятностная система редких событий 
* обработка игровых исходов по матрицам решений 
* отдельное хранилище достижений в JSON 
* загрузка достижений при входе 
* полностью консольный интерфейс
